The Problem

Agents Are Locked Out — and So Is Bitcoin

Agents can research, write, and plan — but they can't buy, sell, or move money.

Premium APIs, digital content, and e-commerce all sit behind human checkout flows. And the world's largest pool of digital capital — Bitcoin, ~$1.5T market cap — has been agent-inaccessible because BTC isn't EVM-native.

x402 Protocol

HTTP 402 Payment Flow — Settled on Mezo

1

Agent Requests Resource

GET /x402/resource/weather-api

2

Server Returns 402

Payment requirements — MUSD amount, chainId 31611, recipient, 15-min expiry

3

Preview Mode (Stops Here)

Returns price to user — "Buy for 0.50 MUSD?"

4

User Confirms → Agent Pays

MUSD ERC-20 transfer on Mezo matsnet, ~1s confirmation

5

Retry with Proof → Content Delivered

X-PAYMENT header, on-chain verification via Mezo RPC, resource served

// 1. Request resource GET /x402/resource/weather-api // 2. Server responds: 402 { "scheme": "spay", "network": "mezo-testnet", "chainId": 31611, "token": "MUSD", "amount": "500000000000000000", "recipient": "0x..." } // 3. Agent transfers MUSD on Mezo await sendMUSD(recipient, "0.50") // tx: 0x9eab1d... (~1s confirmation) // 4. Retry with payment proof GET /x402/resource/weather-api Headers: { "X-PAYMENT": "0x9eab1d..." } // 5. 200 OK — content delivered { "temperature": 72, ... }

Safety & Security

Trust Built Into Every Layer

🔒

Spending Caps

MAX_AUTO_PAYMENT limits per-transaction spend. Default 10 MUSD max.

👁️

Preview Before Pay

Agent always shows price and asks for user confirmation before any payment.

🔑

Non-Custodial

Agent controls its own private key. No third party holds funds.

⛓️

On-Chain Verification

Every payment cryptographically verified against Mezo RPC before content is served.

⏱️

Time-Bounded

15-minute payment windows prevent stale or replayed transactions.

📋

Full Audit Trail

Tx hash, amount, timestamp, recipient — everything logged and verifiable on Mezo Explorer.
